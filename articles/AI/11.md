#  Cursor2.0 /Generate Cursor Rules 命令移除后的替代方案

> Cursor2.0 新增了 Commands 功能， 并移除了 `/Generate Cursor Rules` 一键生成项目规则和更新项目规则的命令， 本文将介绍两种替代方案。

在 Cursor1.0 中， `/Generate Cursor Rules` 可以快速的一键生成符合项目的各种规则分类， 例如前端项目中的 组件规则、api 规则、代码规范规则、组件规范规则等。

![](https://neptune-ipc.oss-cn-shenzhen.aliyuncs.com/img/20251210181134096.png)


但是在 Cursor2.0 中， 移除了 `/Generate Cursor Rules` 命令, 我们可以借助新提供的 `Commands` 功能来自动生成规则。


方案一： 借助NPM包 `generate-cursor-rule` 来创建规则：

1. 全局安装 generate-cursor-rule

   ```bash
    # npm 安装
   npm install -g generate-cursor-rule
   # pnpm 安装
   pnpm add global generate-cursor-rule
   # bun 安装
   bun install -g generate-cursor-rule
   ```

2. 重启 cursor
3.  在 Cursor 中使用命令：/generate-cursor-rule

它的底层原理也很简单， 底层依赖 Commands 功能， 通过命令行调用 generate-cursor-rule 命令来生成规则：

![](https://neptune-ipc.oss-cn-shenzhen.aliyuncs.com/img/20251210181755034.png)
